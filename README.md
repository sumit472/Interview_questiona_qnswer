## 2. What is Java?
Java is an object-oriented programming language developed by Sun Microsystems. 
It is platform independent because it uses JVM (Java Virtual Machine).

## 3. What is JVM?
JVM stands for Java Virtual Machine. It provides runtime environment to execute Java bytecode.
## what is JDK?
JDK is a software development kit used to develop Java applications. It contains JRE, JVM, and development tools like javac, java, and javadoc.
## what is JRE?
JRE is a runtime environment used to run Java applications. It contains JVM and Java class libraries.
## what is Bytecode
Bytecode is an intermediate code generated by the Java compiler from source code, which is executed by the JVM.
## why java is platform independent?
Java is platform independent because the Java compiler converts source code into bytecode, and this bytecode can run on any platform using JVM.
Java Platform Independent क्यों है?
1. Bytecode Generation
Java compiler source code को bytecode (.class) में convert करता है।
2. JVM Execution
हर operating system (Windows, Linux, Mac) के लिए अलग-अलग Java Virtual Machine होती है।
JVM उस bytecode को machine code में convert करके run करती है।
## what is class and object
Class is a blueprint used to create objects, and an object is an instance of a class that represents a real-world entity.
## what is Encapsulation?
Encapsulation is the process of binding data and methods into a single unit and restricting direct access to the data using private variables and public getter/setter methods.
## what is abstraction 
Abstraction is the proc?ess of hiding internal implementation details and showing only the required functionality to the user. In Java it is achieved using abstract classes and interfaces.
## what is inheritance? 
Inheritance is an OOP concept in which one class acquires the properties and methods of another class using the extends keyword. It helps in code reusability.
## what is polymorphism?
Polymorphism means one method having multiple forms. In Java it is achieved by Method Overloading (compile time) and Method Overriding (runtime).
